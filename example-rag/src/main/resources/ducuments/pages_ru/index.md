# Введение

![Проблема интеграции, ширина=300, выравнивание="слева"](spring_ai_logo_with_text.svg)

Проект `Spring AI` нацелен на упрощение разработки приложений, которые включают функциональность искусственного интеллекта без ненужной сложности.

Проект черпает вдохновение из известных Python-проектов, таких как LangChain и LlamaIndex, но Spring AI не является прямым портом этих проектов. Проект был основан с убеждением, что следующая волна приложений генеративного ИИ будет доступна не только для разработчиков на Python, но и будет повсеместно использоваться во многих языках программирования.

> **Примечание:** Spring AI решает основную задачу интеграции ИИ: `Соединение ваших корпоративных данных и API с ИИ моделями`.

![Интерактивный,500,опции=интерактивный](spring-ai-integration-diagram-3.svg)

Spring AI предоставляет абстракции, которые служат основой для разработки ИИ-приложений. Эти абстракции имеют несколько реализаций, что позволяет легко заменять компоненты с минимальными изменениями в коде.

Spring AI предлагает следующие функции:

- Портативная поддержка API для различных поставщиков ИИ для моделей Chat, text-to-image и Embedding. Поддерживаются как синхронные, так и потоковые API. Также доступен доступ к специфическим для модели функциям.
- Поддержка всех основных xref:api/index.adoc[поставщиков ИИ моделей], таких как Anthropic, OpenAI, Microsoft, Amazon, Google и Ollama. Поддерживаемые типы моделей включают:
** xref:api/chatmodel.adoc[Chat Completion]
** xref:api/embeddings.adoc[Embedding]
** xref:api/imageclient.adoc[Text to Image]
** xref:api/audio/transcriptions.adoc[Audio Transcription]
** xref:api/audio/speech.adoc[Text to Speech]
** xref:api/moderation[Moderation]
- xref:api/structured-output-converter.adoc[Структурированные выходные данные] - Сопоставление выходных данных ИИ модели с POJOs.
- Поддержка всех основных xref:api/vectordbs.adoc[поставщиков векторных баз данных], таких как Apache Cassandra, Azure Cosmos DB, Azure Vector Search, Chroma, Elasticsearch, GemFire, MariaDB, Milvus, MongoDB Atlas, Neo4j, OpenSearch, Oracle, PostgreSQL/PGVector, Pinecone, Qdrant, Redis, SAP Hana, Typesense и Weaviate.
- Портативный API для поставщиков векторных хранилищ, включая новый API фильтрации метаданных, похожий на SQL.
- xref:api/tools.adoc[Инструменты/Вызов функций] - Позволяет модели запрашивать выполнение клиентских инструментов и функций, тем самым получая необходимую информацию в реальном времени и предпринимая действия.
- xref:observability/index.adoc[Наблюдаемость] - Предоставляет информацию о связанных с ИИ операциях.
- Ингестация документов xref:api/etl-pipeline.adoc[ETL фреймворк] для Data Engineering.
- xref:api/testing.adoc[Оценка ИИ моделей] - Утилиты для оценки сгенерированного контента и защиты от галлюцинированных ответов.
- Автонастройка Spring Boot и стартеры для ИИ моделей и векторных хранилищ.
- xref:api/chatclient.adoc[ChatClient API] - Удобный API для общения с ИИ моделями чата, идиоматически схожий с API WebClient и RestClient.
- xref:api/advisors.adoc[Advisors API] - Инкапсулирует повторяющиеся паттерны генеративного ИИ, преобразует данные, отправляемые в языковые модели (LLMs) и обратно, и обеспечивает портативность между различными моделями и случаями использования.
- Поддержка xref:api/chatclient.adoc#_chat_memory[Память о беседе] и xref:api/chatclient.adoc#_retrieval_augmented_generation[Увеличенная генерация с использованием извлечения (RAG)].

Этот набор функций позволяет вам реализовывать общие случаи использования, такие как "`Вопросы и ответы по вашей документации`" или "`Чат с вашей документацией.`"

Раздел xref:concepts.adoc[концепции] предоставляет общее представление о концепциях ИИ и их представлении в Spring AI.

Раздел xref:getting-started.adoc[Начало работы] показывает, как создать ваше первое ИИ-приложение. Последующие разделы углубляются в каждую компоненту и общие случаи использования с акцентом на код.

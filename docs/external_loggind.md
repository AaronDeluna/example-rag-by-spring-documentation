# Способы сбора данных о работе веб-приложения SpringBoot
1. Через Actuator:
    - вывести данные о последних 100 запросах через `/actuator/httpexchanges` и `HTTP Trace` (потребуется обработчик body, т.к. по умолчанию его нет)
    - вывести данные через кастомную реализацию `Actuator Endpoints`
2. Шарить данные о работе приложения на отдельных endpont (можно сделать только для тестов):
    - отдавать данные по GET-запросу (если это сделать в тестах, то можно из данных построить диаграмму или веб-страницу с отчетом)
    - создать просмотрщик трейсов, встроенный в приложение, в этом случае не потребуется запускать отдельный OpenTelemetry коллектор
3. Отдавать данные через **JMX Beans**

Все вышеописанные способы относятся к сбору данных только в рамках одного приложения. Для сбора данных по нескольким приложениям потребуется либо создание отдельного коллектора/визуализатора, в который будут собираться все запросы и ответы.
